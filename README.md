# LotWatch

A computer vision system for real-time parking lot occupancy detection. LotWatch processes video footage of a parking lot, classifies each space as occupied or empty on every frame, and writes an annotated output video with colour-coded overlays and a live occupancy counter.


![ParkingOutput](https://github.com/user-attachments/assets/864179ef-a8e1-4bc5-baf5-0ac4f5d42e4e)

---

## How it works

1. **Draw ROIs** — run `draw_rois.py` to manually click the four corners of each parking spot on a still frame. Spots are saved to `ParkingSpots.json`.
2. **Detect** — open `parking_lot_detection.ipynb` and run all cells. The notebook reads the video, applies background subtraction (MOG2) and texture analysis (Laplacian) to each ROI, and classifies every spot per frame.
3. **Output** — an annotated video (`ParkingOutput.mp4`) is written with green overlays for empty spots, red for occupied, and a running count in the corner.

## Usage

### Step 1 — Label parking spots

```bash
uv run python draw_rois.py
```

A window opens on the first frame of `ParkingLot.mp4`. Click the four corners of each parking spot to define its polygon, then use the keyboard controls below. Spots are saved to `ParkingSpots.json` when you press `S`.

| Key | Action |
|-----|--------|
| Left click | Place a corner point |
| Enter | Confirm current 4-point polygon |
| U | Undo last point (or last confirmed spot) |
| S | Save all spots to JSON and exit |
| Q | Quit without saving |

### Step 2 — Run detection

Open the notebook and run all cells:

```bash
uv run jupyter notebook parking_lot_detection.ipynb
```

The notebook will process `ParkingLot.mp4` frame by frame and write `ParkingOutput.mp4`.

## Configuration

All tunable parameters are defined at the top of the notebook:

```python
VIDEO_PATH = "ParkingLot.mp4"
OUTPUT_PATH = "ParkingOutput.mp4"

LAPLACIAN_THRESHOLD = 100    # Higher = stricter occupied detection
MOTION_PIXEL_THRESHOLD = 150 # Min changed pixels to trigger motion alert
MOG2_HISTORY = 150           # Frames used to build background model
MOG2_VAR_THRESHOLD = 50      # MOG2 sensitivity (lower = more sensitive)
PREVIEW_EVERY_N_FRAMES = 12  # Notebook preview frame skip (speeds up rendering)
```

### Tuning tips

- **Missing low-texture cars (e.g. solid red):** lower `LAPLACIAN_THRESHOLD` to ~55–70. Solid-colour cars produce less edge variance in grayscale.
- **Cars absorbed into background over time:** lower `MOG2_HISTORY` to ~60–80. A long history causes stationary vehicles to blend into the background model.
- **Too many false positives:** raise `LAPLACIAN_THRESHOLD` or `MOG2_VAR_THRESHOLD`.

---

## Detection approach

Each frame, for every polygon ROI defined in `ParkingSpots.json`:

- **MOG2 background subtraction** builds a model of the empty lot over `MOG2_HISTORY` frames and flags pixels that deviate from it.
- **Laplacian variance** measures edge/texture density inside the ROI — cars have more edges than bare asphalt.
- A spot is classified as **occupied** if the Laplacian variance exceeds `LAPLACIAN_THRESHOLD`, and as **empty** otherwise.
- Overlays are drawn in **red** (occupied) or **green** (empty), with a total count displayed on each frame.

## Bringing your own video

1. Replace `ParkingLot.mp4` with your footage (or update `VIDEO_PATH`).
2. Re-run `draw_rois.py` to label spots on your new video — this overwrites `ParkingSpots.json`.
3. Run the notebook as normal.

If your video is hosted online, use `yt-dlp` to download it first with ```python uv run yt-dlp -f "best[height<=720]" -o "ParkingLot.mp4" "URL".```
